Who the product thinks you are

Products encode their target user in dozens of small decisions. The interface choice, the data format, the integrations — each one reveals an assumption about who will use this. Reading these signals is a core PM skill.

产品"预设"的用户是谁

产品在无数细节决策中编码了它的目标用户。界面选择、数据格式、集成方式——每一项都隐含着对使用者的假设。读懂这些信号是产品经理的核心技能。

💻
CLI-first interface
The primary interface is a command-line tool. This self-selects for technically literate users comfortable running commands in a terminal.
命令行优先
主要界面是命令行工具，天然筛选出熟悉在终端输入命令的技术型用户。
📁
YAML profile file
Your resume lives in a structured YAML file you edit manually. Non-technical users would find this intimidating — the product doesn't abstract it away.
YAML 简历文件
个人信息存储在需要手动编辑的YAML文件中，非技术用户会觉得门槛较高，产品没有刻意降低这个门槛。
💬
Feishu integration
Feishu is used almost exclusively in Chinese tech companies. This integration signals the builder works in that ecosystem — and built for people like themselves.
飞书集成
飞书几乎专属于中国科技公司。接入飞书强烈暗示开发者本人就在这个生态中工作，并为自己这样的人构建了这款产品。

The two types of job seekers this serves

Based on the product's design signals, it serves two distinct job seeker segments — with different intensities of use.

产品服务的两类求职者

根据产品的设计信号，它服务于两类截然不同的求职者细分群体，使用频率各有差异。

🔥 The Active Hunter 🔥 积极求职者
Job To Be Done 待完成的任务 Apply to 10–20 roles per week while maintaining quality and tracking progress 每周投递10–20个职位，同时保持质量并追踪进度
Pain Point 核心痛点 Tailoring each application manually is exhausting — by application 15, cover letters start sounding identical 手动定制每份申请极度耗力——投到第15份时，求职信开始千篇一律
What They Get 产品给予的价值 Consistent, high-quality output for every application, with history tracking to avoid duplicate efforts 每份申请都能得到高质量的定制材料，历史记录避免重复劳动
🎯 The Strategic Mover 🎯 精准求职者
Job To Be Done 待完成的任务 Evaluate a handful of targeted roles carefully — is this worth applying to? What's my honest fit? 精挑细选少数目标职位——值得投吗？我的真实匹配度是多少？
Pain Point 核心痛点 Hard to self-assess objectively — you either over- or under-estimate your fit without outside analysis 难以客观评估自己——没有外部分析，容易高估或低估与职位的契合度
What They Get 产品给予的价值 An honest match score designed not to inflate — helping prioritize which roles are worth pursuing 不虚高的真实匹配分数——帮助判断哪些职位值得重点追求

One person. One profile. A deliberate choice.

The entire product is designed for a single user. Your profile lives in one file. There's no login, no accounts, no way to share. This isn't an oversight — it's a deliberate scope decision.

单人使用，一个档案——刻意为之

整个产品为单个用户设计。你的档案存在一个文件里，没有登录、没有账户、无法多人共享。这不是疏漏——而是经过深思熟虑的范围决策。

👤 The User Story Behind This 背后的用户故事
The implicit user story: "As a technically literate job seeker, I want to analyze any job description against my own profile instantly, so I can apply to more roles at higher quality without spending hours on manual tailoring."
隐含的用户故事："作为一名有技术背景的求职者，我希望能即时将任何职位描述与我的个人档案进行匹配分析，这样我就能在不花费大量时间手动定制的情况下，以更高质量投递更多职位。" 注意"我的个人档案"——产品由一个人为自己而建，多用户场景被完全推迟。
